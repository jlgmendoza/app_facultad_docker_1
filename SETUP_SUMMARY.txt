🚀 CÓMO INICIAR EN LUBUNTU
═══════════════════════════════════════════════════════════════════════════

OPCIÓN 1: Script interactivo (recomendado para Lubuntu)
  $ chmod +x docker-start.sh
  $ ./docker-start.sh
  ↳ Menú interactivo con 8 opciones

OPCIÓN 2: Con Make (profesional)
  $ make help          # Ver todos los comandos
  $ make up            # Iniciar servicios
  $ make logs          # Ver logs
  $ make shell         # Acceder a bash

OPCIÓN 3: Comandos directos
  $ docker-compose up -d      # Iniciar
  $ docker-compose logs -f    # Logs
  $ docker-compose down       # Detener

📋 GUÍA RÁPIDA DE COMANDOS (Make)
═══════════════════════════════════════════════════════════════════════════

GESTIÓN:
  make up              Iniciar servicios
  make down            Detener servicios
  make restart         Reiniciar servicios
  make ps              Ver estado

DESARROLLO:
  make shell           Bash en la app
  make db-shell        MySQL CLI
  make logs            Ver todos los logs
  make logs-app        Logs de la app
  make logs-db         Logs de MySQL

CONSTRUCCIÓN:
  make build           Reconstruir (sin caché)
  make build-cache     Reconstruir (con caché)

MANTENIMIENTO:
  make clean           Eliminar todo (con volúmenes)
  make prune           Limpiar recursos
  make health          Ver salud de servicios

VERIFICACIÓN:
  make validate        Validar docker-compose.yml
  make env             Crear .env si no existe
  make view-env        Ver variables actuales

🔧 INSTALACIÓN EN LUBUNTU (si no la has hecho)
═══════════════════════════════════════════════════════════════════════════

1. Instalar Docker y Docker Compose
   $ sudo apt-get update
   $ sudo apt-get install -y docker.io docker-compose make

2. Agregar usuario al grupo docker (sin sudo)
   $ sudo usermod -aG docker $USER
   $ newgrp docker

3. Verificar
   $ docker --version
   $ docker-compose --version

❓ SOLUCIÓN DE PROBLEMAS
═══════════════════════════════════════════════════════════════════════════

❌ Permission denied
   ✓ Ejecuta: sudo usermod -aG docker $USER && newgrp docker

❌ Port 3306 already in use
   ✓ Edita .env: DB_PORT=3307
   ✓ O: make clean && make up

❌ Cannot connect to MySQL
   ✓ Verifica: make logs-db
   ✓ Reconstruye: make build && make up

❌ La app no se recarga
   ✓ Verifica que el archivo esté en bind mount
   ✓ Revisa: make logs

💡 TIPS PARA LUBUNTU
═══════════════════════════════════════════════════════════════════════════

• Agrega alias en ~/.bashrc para ahorrar tipeo:
  alias dc='docker-compose'
  alias dmake='make'

• Monitorea contenedores en tiempo real:
  watch -n 1 "docker-compose ps"

• Exporta/importa BD:
  Exportar: make db-shell < backup.sql
  Importar: docker-compose exec -T db mysql -u facultad_user -p < backup.sql

• Usa make help para ver todos los comandos disponibles
